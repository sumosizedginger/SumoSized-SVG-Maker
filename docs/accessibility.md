# SumoSized SVG Generator: Accessibility Standard (WCAG 2.1 AA)

By 2026 standards, web accessibility isn't an afterthought—it's mathematically enforced at the compilation step. The SumoSized engine guarantees compliant vector semantics by default.

## Structural Requirements

Every generated SVG from the `engine/` adheres to the following tree structure:

1. **Root `<svg>` Attributes**: Must contain `xmlns`, `viewBox`, and presentational sizing.
2. **Explicit Title**: A `<title>` element must instantly follow the root open tag, exposing the generator's identity (e.g., _Quantum Core_).
3. **Explicit Description**: A `<desc>` tag must follow the title, outlining the specific parameterized state (e.g., _Complexity 8, Seed 424242_).
4. **`aria-hidden` Safety**: Structural elements devoid of meaning (background boxes, decorative loops) should avoid semantic poisoning. The mathematical generators natively output descriptive nodes.

## High Contrast Mandates

The engine's `getPalette` registry enforces minimum contrast ratios (WCAG \`4.5:1\`) across its color harmonization loops.

- _Color Blindness Matrices_ are supported through the palette overrides.
- _Dark Mode_ is handled natively via transparent backgrounds scaling against `prefers-color-scheme`.

## Reduced Motion Integration

When SVG assets generated by the abstract engines are embedded in web clients, they do not ship with intrinsic CSS animations unless explicitly queried. CSS animations applied downstream should respect the `(prefers-reduced-motion: reduce)` media query.
