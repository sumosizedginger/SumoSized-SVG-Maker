## 3. Findings: The "Hanging" Root Causes

### 3.1 SVG Rasterization (PNG/JPEG/WebM)

The reported `Image rasterization failed` error is the primary blocker.

- **Cause:** The `new Blob()` to `URL.createObjectURL()` flow is hitting security race conditions, likely due to the browser's newly enforced COI context blocking local blob resolution in `img.src`.
- **Solution:** We will pivot to **Base64 Data URIs** for immediate SVG-to-Canvas rendering, bypassing the Blob/URL system entirely.

### 3.2 Video Export (MP4/MOV/WebM)

- **Cause:** FFmpeg software encoding in the main thread (or even an unoptimized worker) is inherently slow and prone to browser-level timeouts.
- **Verified Fact (March 2026):** Mediabunny + WebCodecs API yields up to **50x performance gains** over WASM (e.g., 800fps vs 12fps for MP4 tasks). This eliminates the "All Video Hang" by utilizing hardware acceleration.
