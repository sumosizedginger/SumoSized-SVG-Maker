# SumoSized GIF Maker - Stability & Security Update

## 🚀 Final Technical Debrief

This update resolves the final critical blockers preventing high-performance APNG and GIF exports.

### 🖼️ Slideshow Robustness (Multi-Image Fix)

A silent failure was occurring where only the first slide would be processed.

- **The Issue**: `img.onload` is unreliable for blob URLs that the browser has already parsed for the preview UI—it often skips firing, causing the conversion loop to hang.
- **The Fix**: Implemented the modern `img.decode()` API in `js/modules/conversion.js`. This method returns a promise that resolves once the image is display-ready, regardless of cache state.

### 🎞️ FFmpeg Synthetic Standardization

Fixed a syntax error that caused the FFmpeg engine to reject the secondary encoding pass.

- **The Issue**: Incorrect use of colons (`:`) instead of equals (`=`) for the `paletteuse` filter options.
- **The Fix**: Standardized all `paletteuse` calls to `paletteuse=dither=xxx:diff_mode=xxx`. Also replaced unsupported `format=rgba` with `format=auto` in the overlay chain to prevent engine initialization crashes.

### 🛡️ Secure Telemetry Backend

Updated the Render-hosted API (`RENDER/main.py`) to handle modern cross-origin requests.

- **Changes**: Explicitly white-listed `http://localhost:3000` and `http://127.0.0.1:3000` alongside the production GitHub Pages domain.
- **Benefit**: Local development can now successfully transmit telemetry data without CORS pre-flight rejections.
